## 2025-03-21 - [Missing Authentication and IDOR in Next.js API Routes]
**Vulnerability:** API routes (e.g., `/api/workflows/route.ts`, `/api/links/route.ts`, `/api/corrections/route.ts`, `/api/inbox/[id]/route.ts`) do not require authentication or perform authorization checks to ensure the accessed/modified resource belongs to the currently logged-in user. This allows unauthenticated users to fetch sensitive data, and authenticated users to perform unauthorized actions (Insecure Direct Object Reference) against other users' data.
**Learning:** Next.js API routes in this codebase do not have middleware authentication enabled by default, and `getUser` is not called automatically. Also, the `AuditLog` Prisma model and `Link` model do not have direct `userId` fields to scope authorization checks easily.
**Prevention:** Always explicitly check `getUser()` in every `/api/*` endpoint, ensure a `userId` is available, and scope all Prisma queries using `where: { userId: user.id }` (or appropriate relationships) to verify resource ownership before reading or modifying data.
