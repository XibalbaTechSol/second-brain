## 2024-03-19 - Missing Authentication and IDOR in Dynamic API Routes
**Vulnerability:** The dynamic route `apps/web/src/app/api/entities/[id]/route.ts` processed GET, PUT, and DELETE operations using only the entity ID passed as a URL parameter, without verifying the user's identity or ownership. This allowed full Insecure Direct Object Reference (IDOR) attacks across all entities.
**Learning:** Next.js App Router dynamic routes do not have inherent route-level authentication unless specifically configured via Next.js Middleware. Endpoints fetching data by unique parameter IDs must defensively wrap their handlers with manual authentication (`getUser()`) and verify authorization bounds within the data lookup itself (e.g., scoping `findFirst` to `userId`).
**Prevention:** For any data-modifying operation or sensitive data access on an entity by ID, strictly import `getUser()` and include the user's ID as an access constraint in Prisma (e.g., `findFirst({ where: { id: entityId, userId: user.id } })`). Never trust client-provided IDs without user scope validation.
