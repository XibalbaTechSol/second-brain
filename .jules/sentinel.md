## 2026-03-01 - [Missing Authentication and IDOR in Workflows API]
**Vulnerability:** The workflows API (`/api/workflows`) completely lacked authentication checks (`getUser()`) and allowed any user to update any workflow by supplying an arbitrary `id` (Insecure Direct Object Reference). It also exposed all workflows to anyone via the `GET` endpoint without filtering by the authenticated user's ID.
**Learning:** Next.js route handlers do not automatically enforce authentication. All sensitive endpoints must explicitly call the authentication helper (`getUser()`) and verify the resulting `User | null` object. Additionally, for database operations (`UPDATE`/`DELETE`), we must explicitly check ownership (`userId === user.id`) before allowing modification, rather than assuming the client provided a valid/owned ID.
**Prevention:** Always verify authentication at the start of API route handlers handling sensitive data. When querying or modifying data, enforce tenancy by explicitly including the `userId` in `where` clauses or checking ownership against the database before proceeding.
